# Auto Dev — 사람 게이트 (맥락 우위 잔여)

> 인간은 사용자·제품 맥락을 AI보다 더 잘 안다.  
> 그 격차가 있는 동안만 개입하고, L1 실행·검증은 루프에 맡긴다.

## 게이트 목록

| ID | 이름 | 언제 | 사람 할 일 | 자동이 하면 안 되는 것 |
|----|------|------|------------|------------------------|
| **G1** | 빈틈 승인 | L2 accuracy-defect 클러스터 준비됨 / coverage high-severity | “이 배치를 고칠까?” 예/아니오 | 미승인 빈틈을 coding-fix에 넣기 |
| **G2** | 핵심 파일 merge | `monitor.py` / `streamlit_app.py` 변경 PR | 리뷰 후 merge | 자동 merge, main 직푸시 |
| **G3** | 골든라벨 확정 | 정답지 후보·충돌 | L1 라벨만 확정 (추측 금지) | 라벨 없는 건 모델이 단정 |
| **G4** | 발송·수신자 | 실제 메일 / recipients 변경 | 명시 승인 플래그 | SMTP/Gmail 실발송, 수신자 전체 로그 |

## 개입 최소화 규칙

1. **L1에서는 G1~G4를 호출하지 않는다** (보호파일·발송 위험 시에만 G2/G4로 에스컬레이션).
2. G1은 **건별이 아니라 배치**로 묻는다.
3. “계속할까요?” 중간 확인 금지 — 종료 조건으로만 멈추거나 올린다.
4. BLOCKED는 사람이 원인 해소 후 PENDING으로 **수동** 재투입.

## 자동 구간 체크리스트 (사람 불필요)

- [ ] PENDING 선택·RUNNING 이동  
- [ ] 허용 파일 패치  
- [ ] `loop_verify`  
- [ ] DONE/FAILED/FIX TASK  
- [ ] 상태 파일 커밋 `[skip ci]`  
- [ ] Summary에 루프 5요소·다음 TASK 표시  

이 목록 밖에서 사람이 반복 개입 중이면 → 루프/자산 드리프트로 보고 L2에서 자산을 고친다.
